# docs/ — 기획 & 진행 문서 (세션 이어가기의 근거)

이 폴더는 **터미널이 끊기거나 컨텍스트가 초기화돼도 다음 세션이 그대로 이어받게** 하는 단일 진실이다. 소유: `planner` 역할(다른 역할도 자기 작업 결과를 남긴다).

## 파일 구조

```
docs/
├── README.md        ← 이 규칙
├── PROGRESS.md      ← 지금 상태 (가장 중요, 매번 먼저 읽기)
├── DECISIONS.md     ← 되돌리기 어려운 결정 + 근거 (경량 ADR)
└── plan/            ← 기능별 계획서 (<feature>.md)
```

## 이어가기 프로토콜 (모든 세션/역할 공통)

1. **첫 행동:** `docs/PROGRESS.md`를 읽는다 → `## 진행중`과 `## 다음`을 보고 무엇을 할지 정한다 → 관련 `docs/plan/*.md`를 읽어 맥락을 복원한다.
2. **작업 중:** 의미 있는 결정/가정은 그때그때 해당 `plan` 문서나 `DECISIONS.md`에 적는다. "다음 사람이 추측하지 않게".
3. **진전마다:** `PROGRESS.md`를 갱신하고 **바로 커밋/푸시**한다 (`docs: update progress …`). 크게 쌓지 말고 자주, 작게.
4. **마칠 때:** `진행중` 항목을 `완료`로 옮기고, 다음 세션이 집어들 첫 항목을 `다음` 맨 위에 둔다.

## PROGRESS.md 형식 (고정)

4구획을 항상 유지. 각 항목은 한 줄 + `(plan: docs/plan/x.md)` + `(commit: <해시>)`.

```md
# PROGRESS — <절대날짜 기준>

## 완료 (Done)
- [x] … (commit: abc1234)

## 진행중 (Doing)
- [ ] … — 담당: web-dev — (plan: docs/plan/…)

## 다음 (Next)   ← 다음 세션이 여기 맨 위부터 집어든다
- [ ] …

## 막힘 (Blocked)
- [ ] … — 이유/필요한 것
```

## plan/<feature>.md 형식

상단 프론트매터 + 본문:

```md
---
status: todo | doing | done
last-updated: 2026-07-05
owner: planner
---
# <기능> 계획
## 목표 / ## 사용자 스토리 / ## 데이터모델 / ## 화면 / ## API / ## 작업분해(역할별) / ## 완료 기준
```
